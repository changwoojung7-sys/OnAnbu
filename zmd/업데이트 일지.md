## 1. 보호자(자녀) 메인 홈 화면 UI 개편

- **상단 레이아웃 압축 및 카드화**: 기존에 분리되어 공간을 많이 차지하던 '기상/기분 상태 문구'와 '꽃 이모지' 영역을 가로로 결합했습니다.

  - *보통이에요*: 편안한 녹색(`bg-emerald-50`)
  - *피곤해요/아파요*: 차분한 보라/회색(`bg-purple-50`, `bg-slate-50`)

- **이슈**: 부모님 계정에서 오늘 기분을 선택한 뒤, 마음이 바뀌어 다른 기분을 다시 누르면 보호자 앱이나 DB에 반영(덮어쓰기)되지 않고 무시되는 현상 발견.
- **해결 (`app/parent/index.tsx`)**: 기존 `upsert` 로직에서 고유 복합키 처리가 미흡했던 부분을 보완하여, 저장 전 오늘 날짜의 데이터 존재 여부를 `select`로 먼저 확인하고 **있으면 `UPDATE`**, **없으면 `INSERT`** 하도록 분기 로직을 전면 수정했습니다.
- 이제 부모님 화면에서 언제든 기분을 변경해도 정상적으로 오늘 상태가 업데이트됩니다.

# 2026-03-05 업데이트 일지

오늘(2026년 3월 5일)은 **ON-ANBU (온안부)** 앱의 신뢰성과 데이터 무결성을 확보하기 위해 **회원 탈퇴 프로세스 설계, 고도화 및 기술적 안정화 작업**을 집중적으로 진행했습니다.

## 1. 회원 탈퇴 프로세스 설계 및 문서화

- **전문 문서 작성 (`zmd/회원탈퇴_프로세스.md`)**: 주케어자와 케어대상(부모님) 각각의 역할에 따른 차등 탈퇴 로직과 데이터 파기 권한을 명확히 정의한 전문 설계 문서를 작성했습니다.

## 2. 부모님(케어대상) 탈퇴 옵션 세분화 및 고도화

- **선택형 탈퇴 도입**: 부모님 계정 탈퇴 시 두 가지 옵션을 제공하여 사용자 선택권을 강화했습니다.
  - **옵션 1 (계정만 탈퇴)**: 기기 접속만 차단하고, 그동안 가족들이 정성껏 쌓은 안부/추억 기록은 보존합니다.
  - **옵션 2 (기록 포함 완전 삭제)**: 본인이 전송한 모든 사진, 메시지, 영상을 포함하여 존재 자체를 지우는 옵션입니다.
- **자동 Storage 정리**: 탈퇴 시 수성 저장소(Storage)에 남아있는 미디어 파일(아바타 및 안부 콘텐츠)을 서버 측에서 자동으로 일괄 삭제하여 불필요한 데이터 잔류를 방지했습니다.

## 3. 주케어자(보호자) 탈퇴 UI/UX 개선

- **다단계 확인 모달 구현**: 웹 환경에서 브라우저 기본 알림창(`Alert.alert`)이 작동하지 않거나 차단되는 문제를 해결하기 위해, 앱 내 모달 시스템을 활용한 단계별 확인 UI를 적용했습니다.
- **데이터 연쇄 삭제 (CASCADE)**: 주케어자가 모든 데이터 파기를 선택할 경우, 초대한 부모님의 계정부터 Storage의 모든 가족 미디어까지 한 번에 정리되는 강력한 서버 로직을 구축했습니다.

## 4. 인프라 및 보안 강화 (Edge Functions)

- **안정성 확보**: Edge Function (`withdraw-parent`, `withdraw-guardian`)의 CORS 오류 및 503 크래시 문제를 해결하기 위해 사전 검사(OPTIONS) 로직을 최적화하고 안정화된 라이브러리 로딩 방식을 적용했습니다. (v7 배포 완료)
- **보안 토큰 검증**: 관리자 권한(`service_role`)을 사용하면서도, 실제 요청자가 본인임을 확인하는 수동 JWT 검증 로직을 강화하여 보안을 높였습니다.

## 5. 기타 수정 사항

- 부모님 설정 화면의 회원 탈퇴 플로우와 UI를 주케어자용과 일관성 있게 통일했습니다.
- 탈퇴 프로세스 완료 후 세션을 안전하게 초기화하고 최초 로그인 화면으로 리다이렉트하는 흐름을 최적화했습니다.

# 2026-03-07 업데이트 일지

오늘(2026년 3월 7일)은 **ON-ANBU (온안부)** 앱의 멀티플랫폼 호환성, 스토리지 제약 대응 및 UI 접근성을 개선하기 위한 최적화 작업을 진행했습니다.

## 1. Supabase 스토리지 버킷명 변경 및 하드코딩 제거
- **이슈**: 기존에 사용하던 `media` 버킷 대신 새로운 구조의 `onanbu_media` 버킷으로의 대대적인 이동이 필요.
- **해결**: 프로필 수정, 케어화면, 안부화면, 히스토리 피드 등 앱 전역에 결쳐 하드코딩된 `.from('media')` 및 문자열 분리 로직(`.split()`)을 모두 `onanbu_media`로 안전하게 업데이트했습니다. 

## 2. 모바일 웹 (iOS Safari/Chrome) 대응 및 버그 수정
- **PWA 홈 화면 구 아이콘 표시 (캐싱) 문제**: 아이폰이 홈 화면 추가 시 아이콘(`icon.png`)을 강력하게 캐시하는 현상을 파악하여, 에셋 이름을 `icon_v2.png`로 변경하고 `app.json`에 새로 연결하여(캐시 버스팅) 강제로 새로운 디자인의 아이콘이 표시되도록 수정했습니다.
- **주케어자/케어대상 동영상 업로드 실패 픽스**: 
  - 아이폰 웹 환경에서 대용량 동영상 URL을 `fetch().blob()`으로 다시 읽어들일 때 발생하는 메모리 초과(OOM) 문제를 해결했습니다.
  - ImagePicker가 반환하는 순수 `File` 객체(`asset.file`)를 바로 낚아채서 Supabase 단일 업로드 로직으로 직접 밀어넣도록 `care.tsx`와 `parent/index.tsx`를 전면 수정하여 원활한 동영상 업로드를 보장합니다.

## 3. 스토리지 용량 제한 방어 (50MB Limit)

- **이슈**: Supabase 무료 등급 기본 업로드 제한이 50MB로 설정되어 있어 무거운 동영상 업로드 시 서버 에러 발생.
- **해결**: 부모님 계정과 보호자(자녀) 계정의 업로드 로직 앞단에 **파일 용량 50MB 체크 로직**을 추가하여, 초과 시 즉시 경고 모달 창을 띄우고 앱이 죽거나 서버 오류를 발생시키기 전에 방어하도록 처리했습니다.

## 4. 부모님(케어대상) UI 레이아웃 접근성 강화

- **메인 홈 화면 스크롤 이슈 (`parent/index.tsx`)**: 큰 글꼴(접근성 모드)을 사용하는 부모님 폰이나 작은 디스플레이에서, 중앙의 오늘의 안부 목록이 위아래 엘리먼트에 치여 납작해지는 문제를 해결했습니다. 전체 컨테이너를 스크롤 가능한 `ScrollView`로 변경하여 모든 정보가 안심하고 아래로 확장되게 수정했습니다.
- **설정 화면 겹침 이슈 (`parent/settings.tsx`)**: 화면 최하단에 위치한 '로그아웃'과 '회원탈퇴' 버튼이 홈 인디케이터나 시스템 내비게이션 바에 가려서 안 눌리던 현상을, 하단 패딩(PaddingBottom)을 두 배(`120px`)로 늘려 여유있게 스크롤 되도록 쾌적하게 개선했습니다.

## 5. 온안부 사용자 매뉴얼 추가 및 UI 개편

- **`zmd/manual.md` 완성**: 각 사용자의 앱 내 역할과 권한에 맞춰 (1) 주케어자, (2) 보조케어자, (3) 케어대상으로 나누어 가입/사용법/회원탈퇴 절차를 명확하게 통합 문서화했습니다.

- **케어대상(부모님) 설정 화면 경로 오류 수정**: 부모님 계정으로 접속 시 '설정 > 문의하기' 버튼을 누르면 화면이 깨지던 현상을 발견하여 정상적인 고객센터 테블릿 경로(`/support`)로 우회 연결되도록 라우팅 링크를 수정 조치했습니다.

# 2026-03-08 업데이트 일지

오늘(2026년 3월 8일)은 **ON-ANBU (온안부)** 앱의 UI 레이아웃, 상태 동기화 및 안드로이드 빌드 환경 고도화 작업을 진행했습니다.

## 1. 메인 홈 화면 및 가족 관리 화면 UI 개편 및 렌더링 최적화

- **메인화면(Home) UI 정렬 개선**: 안드로이드 기기 등에서 부모님 프로필(이름과 아바타)이 잘리거나 화면 전체가 과도하게 아래로 처지는 현상을 수정하기 위해, 헤더 컨테이너에 `flex: 1` 레이아웃 속성을 추가하고 `SafeAreaView` 내부 상하단 여백(padding)을 리팩토링했습니다.
- **무한 렌더링(화면 깜빡임) 버그 픽스**: 케어대상을 변경하거나 화면에 포커스가 올 때마다 불필요하게 상태를 중복 덮어씌워 화면이 요동치는 현상을 방지하기 위해, 기존 데이터와 객체 값이 동일한 경우 상태 업데이트를 생략(`JSON.stringify` 비교)하도록 최적화했습니다. 또한 대상을 바꿀 때 이전 사람의 데이터가 일시적으로 남는 화면 잔상 문제를 해결하기 위해 즉시 데이터를 초기화(Clear)하는 로직을 삽입했습니다.
- **가족 관리 화면 스크롤 적용**: 좁은 모바일 환경에서 등록된 부모님이나 가족 수가 많을 경우, 하단 버튼("보조 보호자 초대하기" 등)이나 다른 케어대상이 화면 밖으로 밀려나 보이지 않는 UI 짤림 현상을 해결하기 위해 메인 레이아웃을 `ScrollView`로 전면 교체하여 원활한 스크롤이 가능해졌습니다.

## 2. 케어대상 연동 누락 해결 (데이터베이스 보완)

- 특정 보조 케어자가 초대를 수락했음에도 불구하고, 케어대상 두 분 중 한 분만 보이고 나머지 한 분이 보이지 않던 이슈를 파악 및 해결했습니다. 발급된 초대 코드가 다른 계정에 의해 먼저 소비(수락)되어 발생하는 매핑 누락 문제였습니다. 백엔드(Supabase) `family_members` 테이블에 보조 케어자의 가족 그룹(장모님 그룹) 관계 데이터를 수동으로 직접 복구 기입하여, 관련된 모든 케어 대상이 각 보호자의 앱 화면에 정상적으로 출력되도록 안전하게 연동 조치했습니다.

## 3. 안드로이드 로컬 빌드 환경 동기화

- Expo EAS 클라우드 빌드 한도에 의존하지 않고 로컬 안드로이드 스튜디오 환경에서 쾌적하게 기기 테스트(사이드로드) 및 릴리즈를 진행하실 수 있도록 조치했습니다. `app.json`에 새롭게 지정된 앱 타겟팅 정보(`versionCode: 7`)와 최신 네이티브 서드파티 모듈 의존성들을 명령어(`npx expo prebuild --platform android`)를 통해 로컬의 `android` 빌드 폴더에 온전히 동기화(Sync)시켜 반영을 완료했습니다.
